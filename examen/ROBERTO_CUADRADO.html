Examen Auditoria de Software. Instituto Tecnológico Sudamericano Git y Github

Alumn@: [Nombre Completo]

  1. Qué es un sistema de control de versiones?

    Es una herramienta que ayuda con la gestión de los cambios que se realizan en código o proyecto.

  2. Cuál es la diferencia entre git y github?

    Git es un programa para el versionamiento de software mientras que GitHub es una plataforma donde se guardan repositorios para ayudar al desarrollo colaborativo con versionamiento de software

  3. Cómo creo una rama en git?

    Con el comando git branch "nombre de la rama" o git checkout -b "nombre de la rama" este último hará que git cree la rama y cambie a ella.

  4. Cómo clono un repositorio?

    Con el comando git clone "link del repositorio a clonar"

  5. Qué es la auditoría de software?

    Es un proceso que estudia individual y conjuntamente los sistemas de software que posea una organización para conocer la legalidad y estado de la situación para verificar el correcto uso de estos.

  6. Qué características considera importante para evaluar la facilidad de instalación de un software

    Un aspecto importante sería lo configurable que sea el instalador, ya que mientras más configurable sea se lo puede hacer más intuitivo y amigable con el usuario, haciendo que el usuario tenga menos problemas a la hora de instalar el software.

  7. Qué es un repositorio?

    Es un almacenamiento virtual en el que se pueden guardar las diferentes versiones de un proyecto así como tambien descargarlas.

  8. Cuál es la finalidad de la norma ISO 27000

    Su finalidad es divulgar buenas prácticas para la implementación, mantenimiento y mejora de Sistemas de gestión de seguridad de la información.

  9. Cómo le digo a git cual es mi email y correo?

    Mediante el comando git config --global user.email "correo del usuario".

  10. Cuales son los pasos para conectar git con github?

    Se debe configar el correo y nombre de usuario con los comandos git config --global user.name "usuario de github" y git config --global user.email "correo del usuario" y una vez que intentemos hacer un push hacía un respositorio remoto, git pedirá que nos autentiquemos con nuestras credenciales, una vez hecho esto git quedará vinculado con la cuenta de GitHub.
